# ADR Index

## Purpose

This directory contains architecture decision records for `X-Port`.

Use ADRs to record important technical or structural decisions so future sessions understand not just what was chosen, but why.

---

## When To Add An ADR

Add an ADR when a decision materially affects:

- architecture
- state management
- persistence approach
- testing strategy
- formatting policy
- theming or responsive system structure
- integration patterns

Do not create ADRs for trivial code edits, routine polish, or minor test additions.

---

## ADR Format

Recommended naming:

- `001-short-decision-name.md`
- `002-another-decision.md`

Recommended sections:

- Title
- Status
- Context
- Decision
- Consequences
- Alternatives Considered
